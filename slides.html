Réalisations · 3/4

RapidETNIC S9–S12

Croisement automatisé Rapid7 InsightVM vs ServiceNow CMDB pour détecter les écarts d'inventaire et les assets décommissionnés encore scannés.

fetch_r7.py fetch_snow.py │ │ ▼ ▼ EU Cloud API SNOW REST API (lecture) (lecture) └──────┬───────┘ ▼ fetch.py Matching hostname (IP publique fallback) │ ┌──────┴──────┐ │ │ decomm matched / in_r7 in_r7_only │ On-prem API InsightVM ├─ Tag "deprecated" └─ DELETE agent
Règle de matching
  • Clé primaire : hostname normalisé
  • SNOW asset_tag après le dernier /
  • IP fallback bloqué si plage privée
  • 10.x / 172.16-31.x / 192.168.x = collision VLAN
Actions automatiques
  • Tag deprecated via PUT /api/3/assets/{id}/tags/{tagId}
  • Suppression agent DELETE /api/3/agents/{id}
  • Asset conservé (scans réseau continuent)
Deux API Rapid7

EU Cloud (lecture assets) + On-prem :3780 (write ops)
